# HF-02 — 定义 Public Footprint 公开足迹契约

**Parent:** Home / Feed 定向回流 PRD

**What to build:** 为 Blog、Learn、Projects 向 Feed 产生系统足迹建立统一、可验证的产品契约，使“发布、完成、实质更新”成为公开来时路的唯一系统事件。

**Blocked by:** None — can start immediately.

## Acceptance criteria

- [ ] 系统足迹类型、来源、触发条件、快照字段、去重原则和可见性规则被写成明确需求。
- [ ] Blog 仅在生产部署成功时创建足迹；提交、构建开始、失败和重复部署均不创建。
- [ ] Learn 仅在明确的“小节完成”标记后创建足迹；保存、草稿和普通编辑均不创建。
- [ ] Projects 仅在木下显式标记“实质更新”后创建足迹；系统不做自动推断。
- [ ] 足迹在创建时保存快照；普通源内容编辑不改变足迹。
- [ ] 系统足迹可隐藏但不影响源内容；不做历史回填。
