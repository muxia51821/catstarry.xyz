# Home Activity Signal — Product Requirement Document

> **阶段**：Home Activity Signal 定向 Phase 2（规格化）
>
> **触发原因**：Phase 4.2 星图原型暂停。木下确认 Home 的信号卫星必须真实反映板块是否有公开活动，但 Home 不得恢复内容聚合职责。

---

## Problem Statement

Home 已锁定为宇宙入口与星图导航，不能展示 Recently、混合时间线或跨模块内容。仅有静态星球入口时，访客无法感知哪些板块近期仍有公开活动；但把具体内容带回 Home 又会破坏已锁定的边界。

## Solution

为 Blog、Feed、Learn、Projects 四颗功能星球建立 **Home Activity Signal**：Home 只消费每颗星球的最小活动状态 `active`、`stable` 或 `dormant`，以低音量 Klein Blue 信号卫星表达。About 不参与该模型，豹猫卫星继续只是个人彩蛋。

Home 不得到标题、正文、摘要、链接、列表、时间线、事件数量或访客未读状态；具体公开足迹仍只在 `/feed` 阅读。

## User Stories

1. 作为访客，我想在星图总览中感知某个功能板块近期是否有公开活动，而不被内容卡片打断。
2. 作为访客，我想把活跃、稳定、沉寂理解为板块状态，而不是“我有没有读过”的消息提醒。
3. 作为访客，我想在 Hover、Focus 或进入视野时得到一次克制的信号反馈，而不是持续闪烁的通知点。
4. 作为访客，我想继续前往 `/feed` 查看具体的公开足迹，不在 Home 阅读内容。
5. 作为木下，我发布 Blog、完成 Learn 小节或标记 Projects 实质更新后，想让相应星球真实反映这一公开活动。
6. 作为木下，我发布公开碎碎念或剪藏后，想让 Feed 星球真实反映 Feed 时间线的新活动。
7. 作为木下，我不希望普通编辑、草稿、私密内容、失败部署或重复部署让 Home 发出活动信号。
8. 作为木下，我不希望 About 被误解为需要更新频率的内容板块；豹猫卫星不应成为活动提示。

## Implementation Decisions

- **状态机**：只使用三态。
  - `active`：最近 7 × 24 小时内有符合资格的公开活动。
  - `stable`：超过 7 × 24 小时且不超过 60 × 24 小时有符合资格的公开活动。
  - `dormant`：超过 60 × 24 小时无符合资格的公开活动，或从无符合资格的公开活动。
- **16–60 天处理**：归入 `stable`。它表示星球存在仍可见的活动痕迹，但不再发出近期信号；不引入第四态，也不向 Home 暴露精确年龄或额外强弱等级。
- **来源映射**：
  - Blog：最新公开 Blog Public Footprint。
  - Learn：最新公开 Learn 小节完成 Public Footprint。
  - Projects：最新公开 Projects 实质更新 Public Footprint。
  - Feed：Unified Public Timeline 中最新公开事件，包括原生公开 `note` / `clip` 与公开系统足迹。
  - About：不参与投影。
- **可见性**：只计算当前可公开事件。私密、隐藏、删除或不符合触发规则的事件不能维持或产生活动状态。
- **Home 边界**：Home 只消费受控的最小静态状态投影；它不直接读取 Public Timeline、`feed_posts`、`public_footprints` 或其他模块内容。
- **架构留白**：投影的生成时机、真实数据读取位置、刷新机制、失败降级与 ADR 处理留给定向 Phase 3 裁决。

## Testing Decisions

- 只验证访客可观察的状态结果和 Home 的内容边界，不测试 D1、Worker、构建器或静态文件的内部实现。
- 验证每种合资格事件会影响正确星球；普通编辑、草稿、私密内容、失败部署和重复部署不会影响状态。
- 验证 Feed 的状态以其统一公开时间线为准，而非只看原生帖子或只看系统足迹。
- 验证 Home 任何状态下均不显示具体内容，也不把状态解释为访客未读。

## Out of Scope

- 恢复 `/api/home`、`HomeTimelineItem`、`blog-metadata` KV bridge 或五源内容聚合。
- Home 的 Recently、内容卡片、标题、摘要、列表、筛选、分页或无限流。
- 任何生产代码、Worker、页面路由、canonical CSS 或星球组件实现。
- About 活动状态、豹猫卫星的视觉或交互重做。
- 活动状态的访问者个人化、未读标记、推送或通知中心。
- 历史回填。

## Further Notes

- 这是一次极小定向回流：全局 Phase 2、Phase 3 的既有完成历史不变。
- 定向 Phase 3 必须保持 ADR-006 的核心结论：Home 不恢复聚合 API；推荐优先评估受控静态投影，而非新的 Home 内容接口。
