## 目标

实现 /feed 页面的发布面板：右下角浮动按钮触发弹出面板，支持文字输入、图片/视频选择（互斥），选完文件自动上传到 Cloudflare R2，点击发布后调用 API 创建帖子并即时出现在时间线顶部。

## 验收标准

### 发布面板 UI

- [ ] 右下角浮动按钮（+），点击弹出发布面板，带弹出/收起动画
- [ ] 面板包含：文字输入框 + 媒体选择区 + 发布按钮
- [ ] 未登录时不显示 + 按钮（见 F06 认证登录）

### 媒体选择

- [ ] 统一入口选取文件（不区分图片/视频模式的 UI 切换），系统自动识别类型
- [ ] 选到图片 → 图片模式：最多 6 张，单张 ≤20MB，支持 JPG/PNG/HEIC
- [ ] 选到视频 → 视频模式：只能 1 个，≤100MB 且 ≤60 秒
- [ ] 图片和视频互斥——一条帖子只能选一种媒体类型

### 上传行为

- [ ] 选完文件后立即自动后台上传到 Cloudflare R2（不等用户点发布按钮）
- [ ] 上传中显示进度条 + 百分比数字 + 总文件数标识（如「3/6 张 67%」）
- [ ] 全部上传完成后显示「✓ 完成」，停留 2 秒后消失
- [ ] 逐个删除已选文件，各自独立取消该文件的上传，不影响其他文件
- [ ] 部分文件上传失败：成功的不受影响，失败的原地标记错误 + toast 通知 + 单独重试按钮
- [ ] 上传速度低于 50KB/s 时主动提示「网络较慢，是否继续等待？」
- [ ] 上传中断后自动重试一次，仍失败则提示手动重新选择

### 发布流程

- [ ] 上传未完成时发布按钮禁用，显示「正在上传…」
- [ ] 所有文件上传完成后，发布按钮可点击
- [ ] 点击发布 → 调用 `POST /api/feed` → 成功后关闭面板 → 新帖子出现在时间线顶部
- [ ] 面板关闭（点取消或 X）：未发布的已上传文件在 R2 中保留 12 小时后自动清理（由 F05 Cron Triggers 处理）

## 技术要点

- React island 组件，`client:load` 嵌入 Astro 页面
- 媒体上传到 R2：Phase 3 架构设计阶段确定前端直传（presigned URL）还是通过 Worker 代理上传
- 上传进度使用 `XMLHttpRequest.upload.onprogress` 或 `fetch` + `ReadableStream`
- 文件路径建议：`uploads/feed/{date}/{uuid}.{ext}`

## 测试接缝

- 选图片 → 上传进度显示 → 完成标记 → 选视频（互斥）→ 之前图片被清空
- 选 6 张图 → 其中 1 张上传失败 → 其余 5 张成功 → 失败的单独重试按钮可用
- 上传中点击发布 → 按钮禁用
- 面板关闭重新打开 → 之前未发布的内容不自动恢复（设计决策：B3-2 中木下明确不需要草稿保存）
