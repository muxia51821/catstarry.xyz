## 目标

实现 `/feed` 页面的公开时间线：碎碎念（note）和剪藏（clip）两种卡片混合展示，按时间倒序排列，无限滚动加载。卡片自带类型差异设计，图片和视频有完整的交互体验。

## 验收标准

### 页面结构

- [ ] `/feed` 路由使用 SSR 模式，动态从 Worker API 拉取帖子数据
- [ ] 单列垂直滚动布局：每个帖子卡片结构为 文字 → 媒体 → 时间戳
- [ ] 内容互斥：纯文字 / 纯图片 / 纯视频，三种不可在同一帖中混排

### 图片展示

- [ ] 缩略图使用 1:1 正方形裁剪
- [ ] 1 张图：全宽展示
- [ ] 2 张图：并排各 50%
- [ ] 3 张图：并排各 33.3%
- [ ] 4 张图：2×2 宫格
- [ ] 5 张图：上排 3 张 + 下排 2 张（3+2）
- [ ] 6 张图：2×3 网格

### 图片弹窗

- [ ] 点击图片打开弹窗：半透明黑底覆盖全屏
- [ ] 弹窗内仅显示图片，无额外控件（无关闭按钮、无工具栏）
- [ ] 支持平滑滑动翻页（触摸 swipe / 鼠标拖拽）
- [ ] 方向键（← →）翻页
- [ ] 点击图片外部区域关闭弹窗
- [ ] 弹窗内图片最大高度 = 视口高度的 61.8%

### 视频展示

- [ ] 视频缩略图保留原始宽高比
- [ ] 缩略图上叠加半透明 ▶ 播放图标
- [ ] 点击切换播放/暂停
- [ ] 手机端：长按播放，松手停止
- [ ] 电脑端：点击切换播放/暂停

### 无限滚动

- [ ] 滚动到页面底部时自动加载下一页（游标分页）
- [ ] 加载中显示加载指示器
- [ ] 没有更多内容时显示「没有更多了」

### 剪藏卡片

- [ ] 剪藏卡片含有：标题（可点击跳转原文）、摘要、封面缩略图、用户评论
- [ ] 视觉风格与碎碎念卡片可区分

## 技术要点

- Astro SSR 页面 + React island 做客户端交互（弹窗、视频播放、滚动加载）
- 游标分页：前端存 `nextCursor`，滚动到底触发 `GET /api/feed?cursor=xxx`
- 图片弹窗和视频播放器为独立 React 组件

## 测试接缝

- 访问 `/feed` → 确认 SSG/SSR 输出包含帖子数据
- 滚动到底部 → 确认新帖子加载
- 点击图片 → 弹窗出现 → 滑动翻页 → 点击外部关闭
- 点击视频 → 开始播放 → 再次点击暂停
- HEIC 图片在 Firefox 中可能显示空白——需求分析中木下接受此风险，不做额外处理
