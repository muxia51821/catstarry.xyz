## 目标

实现 `/feed/admin` 管理后台页面（仅登录后可见）：统一管理原生帖子与系统足迹。原生帖子支持隐藏/恢复/物理删除；系统足迹仅支持隐藏/恢复，保留审计快照。

## 验收标准

### 页面路由与权限

- [ ] `/feed/admin` 路由，仅登录后可见（未登录重定向到 `/feed`）
- [ ] 管理后台为独立页面，不在时间线中掺杂管理 UI（需求决策：B3-2 第 2 层，木下坚持只要管理页面不要卡片快捷入口）

### 帖子列表

- [ ] 帖子列表按时间倒序排列
- [ ] 每条条目显示摘要信息：内容前 50 字或系统快照、类型（碎碎念/剪藏/Blog 发布/Learn 完成/Projects 更新）、发布时间、当前状态（公开/仅我可见）
- [ ] 列表支持分页（每页 20 条）

### 筛选

- [ ] 按状态筛选：公开 / 仅我可见 / 全部（默认）
- [ ] 按类型筛选：碎碎念 / 剪藏 / 系统足迹（可继续按来源模块细分）/ 全部（默认）
- [ ] 按日期范围筛选（起止日期）
- [ ] 多个筛选条件可组合使用

### 多选批量操作

- [ ] 每条帖子前有勾选框，支持全选/取消全选
- [ ] 批量隐藏：选中多条 → 点击「设为仅我可见」→ 确认弹窗 → 帖子状态变为 private → 时间线上消失
- [ ] 批量恢复：选中已隐藏的帖子 → 点击「恢复公开」→ 帖子重新出现在时间线上
- [ ] 批量删除只对原生帖子可用；选中系统足迹时不显示删除操作

### 单条操作

- [ ] 原生帖子右侧有「⋮」菜单：隐藏/恢复公开、删除；系统足迹只有隐藏/恢复公开
- [ ] 删除操作必须二次确认弹窗

### 不可逆约束

- [ ] 删除后的原生帖子不可恢复（需求硬约束）
- [ ] 原生帖子物理删除（`DELETE FROM feed_posts WHERE id = ?`）；系统足迹不得通过此端点物理删除
- [ ] 隐藏系统足迹不会影响 Blog、Learn 或 Projects 的源内容

## 技术要点

- Worker API 新增：
  - `GET /api/feed/admin?status&type&from&to&cursor`（管理专用查询，包含私密帖子）
  - `PATCH /api/feed/:id`（更新 visibility 字段）
  - `DELETE /api/feed/:id`（仅物理删除原生帖子）
  - 系统足迹 visibility 更新走受限的 footprint 管理路径，不暴露为来源内容写入接口
- 管理后台前端为独立 Astro 页面 + React island 做交互
- 筛选条件通过 query string 传递，前端不维护复杂状态

## 测试接缝

- 登录后访问 `/feed/admin` → 看到全部帖子列表（含私密帖子）
- 按「仅我可见」筛选 → 列表只剩已隐藏帖子
- 勾选 3 条帖子 → 批量隐藏 → 确认 → 列表刷新 → 状态变为 private
- 勾选 2 条已隐藏帖子 → 恢复公开 → 去 `/feed` 确认时间线上重新出现
- 勾选 1 条帖子 → 删除 → 二次确认 → 确认 → 帖子消失 → 刷新不再出现
