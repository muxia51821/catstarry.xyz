# 学习笔记验收清单

> 木下验收用。按实际操作流程组织。不说技术术语。

## 1. 浏览学习笔记首页

- [ ] 打开 /learn，看到所有学习track的列表
- [ ] 每个track显示：名称、课程数量、最后更新时间
- [ ] 顶部有一个知识图谱图（静态图），鼠标悬停高亮对应轨道，点击跳转
- [ ] 有一个搜索框，输入关键词实时筛选笔记

## 2. 按track查看笔记

- [ ] 点击某个tack，看到该轨道下的所有笔记
- [ ] 轨道内部支持分节（子分类），通过 tab 切换或无限嵌套层级

## 3. 阅读笔记详情

- [ ] 点击一篇笔记进入详情页，看到标题、发布日期、标签、正文
- [ ] 正文渲染正常（代码块、图片、表格、链接）
- [ ] 文中有 [[另一篇笔记]] 格式的链接，点击能跳转
- [ ] 鼠标悬停在 wikilink 上，弹出预览卡片（标题+摘要）
- [ ] 页面右侧有目录树，显示当前笔记在知识体系中的位置，当前笔记高亮

## 4. 搜索笔记

- [ ] 在搜索框输入关键词，结果实时筛选
- [ ] 搜索匹配标题（模糊）、标签（精确）、track名（精确）
- [ ] 输入时有自动补全建议
- [ ] 没有匹配时显示「没有找到相关笔记」

## 5. 草稿和发布管理

- [ ] 通过 teach skill 生成的笔记默认为草稿状态
- [ ] 草稿笔记在 /learn 首页、搜索结果、Feed、RSS 中都不可见
- [ ] 直接访问草稿笔记的 URL 显示 404
- [ ] 进入管理后台（需登录），看到所有笔记列表（含草稿）
- [ ] 点击「发布」，笔记立刻在 /learn 中可见；发布本身不产生 Feed 足迹

## 6. 完成小节后的 Feed 足迹

- [ ] 我明确标记“完成一个小节”后，Feed 出现一条「Learn 完成小节」足迹
- [ ] 足迹显示该小节创建时的标题、摘要、时间和前往笔记的链接
- [ ] 保存、普通编辑、草稿和仅发布笔记不会产生足迹
- [ ] 同一小节的同一次完成不会因重复保存产生多条足迹
- [ ] 隐藏该条足迹不会隐藏或删除原笔记

## 7. RSS 订阅

- [ ] 博客和学习笔记统一在一个 RSS feed 中
- [ ] 只包含已发布内容，草稿不在其中

## 8. 手机浏览

- [ ] 用手机打开 /learn，首页排版正常
- [ ] 目录树变为抽屉式，点击汉堡菜单打开
- [ ] 知识图谱在手机上隐藏或缩小为缩略图
- [ ] 笔记详情页的 wikilink 预览适配触屏操作
