# 项目展示验收清单

> 木下验收用。按实际操作流程组织。不说过多技术术语。

## 1. 浏览项目页面

- [ ] 打开 /projects，看到项目卡片网格布局
- [ ] 每张卡片显示：项目截图、项目名称、一句话描述、技术标签
- [ ] 点击卡片跳转到项目的外部链接

## 2. 项目实质更新后的 Feed 足迹

- [ ] 我在项目发生实质更新时可显式标记它；系统不会根据普通编辑自动猜测
- [ ] 标记后，Feed 出现一条「Projects 实质更新」足迹，带项目名称、摘要、时间和链接
- [ ] 普通索引编辑、保存和重复部署不会产生足迹
- [ ] 隐藏该条足迹不会隐藏或删除 /projects 中的项目

## 3. 新项目上架

- [ ] 我告诉 AI agent 有新项目要上架
- [ ] AI agent 按模板添加项目索引信息
- [ ] Git push 后自动部署，新卡片出现在 /projects
