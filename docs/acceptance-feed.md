# Feed 公开足迹验收清单

> Feed 是木下的公开足迹／来时路。它混排原生内容与重要系统事件，但不是社交平台，也不回填历史。

## 1. 原生发布

- [ ] 木下可发布碎碎念：文字，可选图片或视频。
- [ ] 木下可发布剪藏：粘贴链接后得到标题、摘要、封面图，并可补充自己的点评。
- [ ] 原生发布面板只提供碎碎念和剪藏，不提供“伪造 Blog 发布 / Learn 完成 / Projects 更新”的入口。
- [ ] 发布成功后，原生内容立刻出现在 Feed 时间流。

## 2. 公开足迹类型

- [ ] Feed 按时间倒序混排：碎碎念、剪藏、Blog 发布、Learn 小节完成、Projects 实质更新。
- [ ] 系统足迹清楚显示来源与动作，例如“Blog 发布”“Learn 完成小节”“Projects 实质更新”。
- [ ] 系统足迹的视觉表达克制、可辨认，不伪装成木下手写的碎碎念。
- [ ] 每条系统足迹可进入其来源内容；来源暂不可用时，仍保留创建时的标题、摘要和时间快照。

## 3. 系统足迹触发

- [ ] Blog 只有在生产部署成功后产生一条足迹；提交、构建开始、部署失败或重复部署不产生。
- [ ] Learn 只有在明确标记“完成一个小节”后产生一条足迹；草稿、保存、普通编辑不产生。
- [ ] Projects 只有在木下显式标记“实质更新”后产生一条足迹；系统不自动猜测。
- [ ] 同一来源版本不会产生重复足迹。
- [ ] 不回填上线前的 Blog、Learn、Projects 或 Poker 历史。

## 4. 快照与可见性

- [ ] 系统足迹在创建时保存标题、摘要、链接、时间及来源标识。
- [ ] 普通编辑来源内容后，既有足迹不被自动改写。
- [ ] 木下可在管理后台隐藏系统足迹；隐藏后它不在公开 Feed 出现。
- [ ] 隐藏系统足迹不隐藏、不删除或不改变其来源内容。

## 5. 浏览与媒体

- [ ] 原生碎碎念卡片保留文字、图片网格或视频封面的浏览体验。
- [ ] 剪藏卡片保留标题、摘要、封面缩略图和木下点评。
- [ ] 点击图片可放大查看；点击视频可播放或暂停。
- [ ] 时间流可持续浏览更多较早内容。

## 6. 登录和管理

- [ ] 未登录访客只能浏览公开 Feed。
- [ ] 木下登录后可发布原生内容，并进入独立管理页面。
- [ ] 管理页面可按可见性、事件类型和日期筛选原生内容与系统足迹。
- [ ] 原生内容仍可隐藏或删除；系统足迹至少可隐藏，且不会影响来源内容。

## 7. 设备与边界

- [ ] 手机、平板和桌面端均能舒适浏览 Feed。
- [ ] Feed 不提供评论、点赞、多用户发布、定时发布或历史回填。
