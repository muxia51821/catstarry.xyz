# 博客验收清单

> 木下验收用。按实际操作流程组织。不说技术术语。

## 1. 写文章和发布

- [ ] 我在 content/blog/ 下新建一个 Markdown 文件，填写标题、日期、分类、标签、摘要
- [ ] 我把文章推到 GitHub，几分钟后博客自动更新，新文章出现在列表里
- [ ] 文章 URL 是 /blog/文章slug/，简洁可读
- [ ] 我把 draft: true 的文章推上去，它不会出现在网站里

## 2. 浏览博客列表

- [ ] 打开 /blog，看到所有已发布文章按时间倒序排列
- [ ] 每篇文章卡片显示：标题、日期、分类、标签、摘要、阅读量
- [ ] 文章超过 10 篇后出现分页导航（上一页/页码/下一页）
- [ ] 侧栏有分类列表（技术/生活/观点，显示文章数量）和标签云

## 3. 阅读文章

- [ ] 点击文章标题进入详情页，看到标题、日期、分类、标签、阅读量
- [ ] Markdown 正文渲染正常（标题层级、代码高亮、图片、表格、链接）
- [ ] 文章底部有分享按钮（复制链接、X/Twitter、微信）
- [ ] 文章底部有 Giscus 评论区，能用 GitHub 账号评论
- [ ] 每刷新一次页面，阅读量 +1（同一天同一个人不重复计数）

## 4. 按分类或标签找文章

- [ ] 点击侧栏某个分类（如「技术」），跳转到 /blog/category/tech/，只显示该分类的文章
- [ ] 点击某个标签（如「Astro」），跳转到 /blog/tag/astro/，只显示该标签的文章
- [ ] 分类页和标签页也支持分页

## 5. RSS 订阅

- [ ] 在 RSS 阅读器中添加 /blog/rss.xml，能看到所有已发布文章

## 6. 手机浏览

- [ ] 用手机打开博客，排版正常，字号舒适，阅读不费眼
